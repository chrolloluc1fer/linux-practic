1. **UID (User ID)**
    
    - У каждого пользователя в Linux есть числовой идентификатор (UID). Обычно для системных пользователей UID маленький (до 1000), а для обычных — начинается с 1000 или 500 (зависит от дистрибутива).
    - Зная UID, система понимает, под каким именем запущен процесс или обрабатывается файл.
2. **GID (Group ID)**
    
    - Похожая логика действует и для групп. У каждой группы есть числовой идентификатор (GID).
    - Пользователь может состоять в нескольких группах — основной (primary group) и дополнительных (secondary groups).
3. **/etc/passwd и /etc/shadow**
    
    - Файл `/etc/passwd` хранит основные данные о пользователях (логин, UID, GID, домашняя директория, используемый shell).
    - Файл `/etc/shadow` содержит зашифрованные пароли и информацию о политике паролей (срок действия, дата последней смены и т.д.).

---

Права `rwx` для директорий работают немного иначе, чем для файлов:

1. **r (read)**
    
    - Разрешает просматривать список содержимого директории (имена файлов).
    - Без `x` (execute) вы видите список имён, но не сможете «войти» в каталог (cd).
2. **w (write)**
    
    - Позволяет создавать, удалять и переименовывать файлы внутри директории.
    - Если у вас есть `w`, но нет `r`, вы можете добавлять/удалять файлы, не видя полного списка.
3. **x (execute)**
    
    - Разрешает «вход» в директорию (команда `cd`).
    - Без `r` и `x` вы не сможете даже проверить, что внутри, а без `x` не сможете перейти в каталог, даже если вы видите его содержимое.

### 2.1 `groupadd`

**Назначение**  
Создаёт новую группу в системе. Обычно указывается имя группы; при необходимости можно задать определённый GID (Group ID).

**Синтаксис**

```css
groupadd [опции] <имя_группы>
```

- **Пример**:
    
    ```nginx
    groupadd devteam
    ```
    
    Создаёт группу `devteam` с автоматически сгенерированным GID.

### 2.2 `groupdel`

**Назначение**  
Удаляет группу из системы. Применяется, когда группа больше не используется или когда нужно избежать накопления «мертвых» записей.

**Синтаксис**

```php-template
groupdel <имя_группы>
```

- **Важно**: перед удалением стоит проверить, нет ли пользователей, чьей основной группой является данная. Если она основная, необходимо сначала сменить основную группу пользователей (через `usermod`).

### 2.3 `gpasswd`

**Назначение**  
Управляет паролем группы и её членами. Когда группа защищена паролем, пользователи могут временно присоединяться к ней через команду `newgrp`, введя пароль. Это более старая модель, которая реже используется, но иногда встречается.

**Пример**

```nginx
gpasswd devteam
```

Попросит установить пароль для группы `devteam`.

Кроме того, `gpasswd -a <user> <group>` добавляет пользователя в группу, а `gpasswd -d <user> <group>` удаляет пользователя из группы.

### 2.4 `usermod -aG`

**Назначение**  
Позволяет добавить пользователя в дополнительную (вторичную) группу, не затрагивая его основную группу и прочие настройки.

**Синтаксис**

```php-template
usermod -aG <имя_группы> <имя_пользователя>
```

- **Опция `-a`** (append) означает «добавить пользователя в указанные группы, не убирая его из других». Без `-a` пользователь будет исключён из всех групп, кроме перечисленных.
- **Пример**:
    
    ```nginx
    usermod -aG devteam alice
    ```
    
    Пользователь `alice` получит дополнительный доступ, который есть у группы `devteam`.
